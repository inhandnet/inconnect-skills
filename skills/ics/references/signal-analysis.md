# 信号质量分析

蜂窝路由器的信号质量评估与分析指南。

## 查询信号数据

```bash
ics router signal <id> --after <start> --before <end> --oid <oid>
ics router signal <id> --fields rssi,rsrp,sinr --after <start> --oid <oid>
```

建议指定时间范围，避免返回过多历史数据。省略时间则回退到该端点默认窗口（约近 7 天）。

## 关键信号指标

### RSSI（接收信号强度指示）

| 范围 | 等级 | 说明 |
|------|------|------|
| ≥ -65 dBm | 优秀 | 信号极强 |
| -65 ~ -75 dBm | 良好 | 满足大多数应用 |
| -75 ~ -85 dBm | 一般 | 可用但可能有波动 |
| -85 ~ -95 dBm | 较差 | 丢包率上升，速率下降 |
| < -95 dBm | 极差 | 频繁断线 |

### RSRP（参考信号接收功率，4G/5G）

| 范围 | 等级 |
|------|------|
| ≥ -80 dBm | 优秀 |
| -80 ~ -90 dBm | 良好 |
| -90 ~ -100 dBm | 一般 |
| -100 ~ -110 dBm | 较差 |
| < -110 dBm | 极差 |

### SINR（信号与干扰加噪声比，4G/5G）

| 范围 | 等级 |
|------|------|
| ≥ 20 dB | 优秀 |
| 13 ~ 20 dB | 良好 |
| 0 ~ 13 dB | 一般 |
| < 0 dB | 极差（干扰严重） |

### RSRQ（参考信号接收质量）

| 范围 | 等级 |
|------|------|
| ≥ -10 dB | 良好 |
| -10 ~ -15 dB | 一般 |
| < -15 dB | 较差 |

## 分析要点

1. **看趋势而非单点**：信号会因环境变化波动，关注整体趋势
2. **多指标综合判断**：RSSI 好但 SINR 差说明有干扰；RSRP 差但 SINR 好说明是覆盖弱
3. **对比时间段**：故障时段 vs 正常时段的信号对比
4. **关注骤降**：信号突然大幅下降通常意味着天线松动、SIM 卡接触不良或基站切换

## 常见问题与建议

| 现象 | 可能原因 | 建议 |
|------|----------|------|
| RSSI 持续低于 -90 | 覆盖弱 | 更换天线位置、使用高增益天线 |
| SINR 很低但 RSSI 正常 | 干扰 | 更换频段、调整天线方向 |
| 信号周期性波动 | 环境因素 | 观察是否与时间/天气相关 |
| 信号突然消失 | 硬件问题 | 检查天线连接、SIM 卡 |

> 信号差导致 VPN 频繁掉线时，结合 `ics router online-trend` 对比信号低谷与掉线时刻。
